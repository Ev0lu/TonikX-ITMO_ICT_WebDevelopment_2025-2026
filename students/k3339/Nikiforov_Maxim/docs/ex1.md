# Задание 1 — UDP эхо

## Цель
Освоить работу с UDP‑сокетами: отправка/приём датаграмм без установления соединения.

## Протокол и API
- Адресное семейство: `AF_INET` (IPv4)
- Тип сокета: `SOCK_DGRAM` (UDP)
- Ключевые вызовы:
  - Сервер: `socket()`, `bind()`, `recvfrom()`, `sendto()`
  - Клиент: `socket()`, `sendto()`, `recvfrom()`

## Файлы
- `Lr1/ex1/server.py`
- `Lr1/ex1/client.py`

## Запуск
1. Терминал 1:
   - `cd ...\Lr1\ex1`
   - `python server.py`
2. Терминал 2:
   - `cd ...\Lr1\ex1`
   - `python client.py`

Ожидаемый результат: сервер выводит полученное сообщение и отвечает «Hello, client»; клиент печатает ответ.

## Важные нюансы
- UDP не устанавливает соединение; каждое сообщение — отдельная датаграмма, границы сообщений сохраняются.
- Сервер не должен закрывать сокет внутри цикла. Иначе следующая итерация `recvfrom()` вызовет ошибку наподобие `WinError 10038`.  
  Решение: не закрывать сокет в цикле, либо завершать цикл (`break`) после одноразового ответа.
